2022 年伊朗全国断网期间，Telegram 在伊朗的访问量不降反升。原因很简单——MTProto 代理。大量志愿者在世界各地搭建了 MTProto 代理服务器，帮助伊朗用户绕过网络封锁连接到 Telegram。

这背后支撑一切的核心技术，就是 MTProto。它不是某个商业 VPN 协议，而是 Telegram 团队从零设计、专为移动互联网优化的加密通信协议。本文将全面解读 MTProto 的工作原理、安全特性和代理使用方法。

## 一、MTProto 是什么？

### 1.1 一句话定义

MTProto（Mobile Protocol）是 Telegram 自研的加密通信协议，用于客户端与服务器之间以及客户端与客户端之间的所有数据加密传输。可以把它理解为 Telegram 的「内部语言」——所有 Telegram 客户端发出的数据，都被 MTProto 封装成加密数据包。

### 1.2 为什么需要自研协议？

市面上已有 TLS/SSL（HTTPS 用的那个）、IPsec、WireGuard 等成熟加密协议，Telegram 为什么不直接用？

**移动网络特殊需求：**
- 蜂窝网络频繁切换（4G↔Wi-Fi↔5G），传统协议的重连延迟太高
- 移动设备电量敏感，加密计算必须极度省电
- 弱网环境（2G/EDGE）下需要工作，数据包必须尽量小

**Telegram 的独特需求：**
- 需要支持异步消息（离线消息的加密存储和转发）
- 需要同一协议同时支持推流（消息推送）和拉流（主动获取）
- 需要内建抗审查机制（伪装流量、代理中继）

### 1.3 MTProto 的发展历程

| 版本 | 推出时间 | 关键变化 |
|------|---------|---------|
| MTProto 1.0 | 2013 | 初始版本，支持基本加密传输 |
| MTProto 2.0 | 2017 | SHA-256 替代 SHA-1，修复已知漏洞 |
| 当前版本 | 持续更新 | 安全加固、性能优化 |

目前所有 Telegram 客户端都使用 MTProto 2.0。旧版本早已废弃，不存在降级攻击的风险。

## 二、MTProto 的加密结构

### 2.1 三层加密模型

MTProto 采用了独特的三层加密结构：

```
┌─────────────────────────┐
│   应用层（消息内容）       │  ← 实际传输的数据
├─────────────────────────┤
│   MTProto 加密层          │  ← AES-256-IGE 加密
├─────────────────────────┤
│   传输层（TCP/HTTP/WebSocket）│  ← 底层网络
└─────────────────────────┘
```

### 2.2 加密层核心算法

| 组件 | 算法 | 作用 |
|------|------|------|
| 对称加密 | AES-256-IGE | 加密消息体 |
| 密钥派生 | KDF（基于 SHA-256） | 从 auth_key 派生 msg_key |
| 完整性校验 | HMAC-SHA256 | 检测消息是否被篡改 |
| 密钥协商 | Diffie-Hellman 2048-bit | 客户端与服务器建立共享密钥 |

### 2.3 auth_key 的生成过程

MTProto 的安全性建立在 auth_key（授权密钥）上。其生成过程设计精巧：

1. 客户端生成一个随机 nonce
2. 客户端向服务器发送 `req_pq` 请求，获取服务器的 RSA 公钥和质数
3. 客户端生成 DH 参数，用服务器公钥加密后发送
4. 服务器解密后完成 DH 密钥交换
5. 双方各自计算出相同的 auth_key

这个 auth_key 有两个重要作用：
- 它是后续所有消息加密的基础材料
- 它永久绑定该设备和 Telegram 账号

一旦你换了设备或者退出登录重新登录，之前的 auth_key 就作废了——这自然提供了设备级别的安全隔离。

### 2.4 msg_key 防重放攻击

每条消息都有一个独特的 msg_key，由消息内容和部分 auth_key 派生的 SHA-256 哈希生成。如果攻击者截获了一条加密消息并试图重放它（原封不动地再发一次），服务器会检测到 msg_key 已失效并拒绝处理。

这个机制防止了一种常见的攻击方式：重放攻击（Replay Attack）。

## 三、MTProto 代理：原理、类型和选择

### 3.1 什么是 MTProto 代理？

MTProto 代理是 Telegram 官方支持的代理类型。它在 Telegram 客户端和 Telegram 服务器之间充当加密中继点。

```
正常连接: 客户端 ──MTProto──→ Telegram 服务器
代理连接: 客户端 ──MTProto──→ MTProto Proxy ──转发──→ Telegram 服务器
```

关键点：MTProto 代理**看不到**消息内容。因为代理只是在转发已经用 MTProto 加密的数据包。代理服务器看到的和你 ISP 看到的一样——全是密文。

### 3.2 三种代理类型对比

Telegram 支持多种代理类型，各有不同的用途：

| 代理类型 | 加密方式 | 伪装能力 | 用途 | 推荐度 |
|---------|---------|---------|------|--------|
| MTProto Proxy | MTProto | 可伪装成 TLS | 绕过封锁 | ⭐⭐⭐⭐⭐ |
| SOCKS5 | 无额外加密 | 依赖上层 | 通用代理 | ⭐⭐⭐ |
| HTTP | 无额外加密 | 有限 | 兼容旧环境 | ⭐⭐ |

MTProto 代理又分为两种模式：

**内置代理（内置 MTProto Proxy）：**
- 代理头部添加随机填充，使流量看起来像随机数据
- 无法通过深度包检测（DPI）的情况下效率最高
- 需要代理服务器运行 `mtproto-proxy` 官方软件

**伪装代理（Fake TLS）：**
- 在 MTProto 流量外层包裹一层假的 TLS 握手
- 让流量看起来像是普通的 HTTPS 流量（访问网站）
- 可以绕过大多数基于 DPI 的审查系统
- 配置时在代理地址后添加 `ee` 前缀

### 3.3 如何选择代理模式？

**如果你的网络环境只是屏蔽了 Telegram 服务器 IP：** 使用内置代理模式就够了。

**如果你的网络环境有深度包检测（DPI），会分析流量特征：** 必须使用伪装 TLS 模式。中国、伊朗、俄罗斯等国家的网络环境通常需要这个模式。

**如果你不确定：** 先用内置模式试试，不行再换伪装模式。

## 四、客户端连接 MTProto 代理

### 4.1 获取代理地址

MTProto 代理的地址格式通常是：

```
代理服务器IP:端口
```

或者带伪装密钥的格式：

```
ee443a2b...加密密钥...@代理服务器IP:端口
```

获取代理地址的几种方式：
- 朋友/同事分享（最可靠）
- Telegram 公开代理频道
- 自己搭建（下文会讲）
- MTProxy 公共服务列表（质量参差不齐）

**安全提示：** 不要使用来路不明的免费代理。代理虽然看不到你的消息内容，但可以看到你的连接元数据（登录时间、流量大小），这些信息配合 IP 地址可能暴露你的大致位置和使用习惯。

### 4.2 Android 端连接步骤

1. 打开 Telegram，点击左上角菜单（≡）
2. 进入「设置」
3. 找到「数据和存储」→「代理设置」
4. 点击「添加代理」
5. 选择代理类型为「MTProto」
6. 输入服务器地址、端口和密钥（Secret）
7. 点击保存

连接成功后，Telegram 顶部状态栏会显示一个盾牌图标，表示当前在通过代理连接。

### 4.3 iOS 端连接步骤

1. 打开 Telegram → 设置 → 数据和存储
2. 选择「代理」→ 「添加代理」
3. 类型选择「MTProto」
4. 输入服务器、端口、密钥
5. 点击保存，返回后确保代理已启用

### 4.4 桌面端连接步骤

1. Telegram Desktop → 设置 → 高级
2. 找到「连接类型」→ 「使用自定义代理」
3. 选择「MTProto」
4. 输入代理信息
5. 保存

桌面端的代理设置会记住最近使用的代理，切换比较方便。

### 4.5 连接不上怎么办？

代理连不上的常见原因和排查步骤：

- **基础连通性：** 先用 `ping` 命令测试代理服务器 IP 能否 ping 通
- **端口问题：** 确认端口没有被本地防火墙拦截
- **密钥错误：** MTProto 密钥是 32 字节的十六进制字符串，复制粘贴时多一个空格或少一个字符都会导致连接失败
- **代理服务器宕机：** 如果你用的是公共代理，可能已经下线了；多备几个地址
- **网络策略变化：** ISP 可能更新了封锁策略，试试换伪装 TLS 模式

## 五、自行搭建 MTProto 代理

### 5.1 为什么自己搭？

- 安全：代理服务器在你自己的 VPS 上，不会被第三方记录流量元数据
- 稳定：不会因为公共代理被滥用而封 IP
- 可控：何时关停、何时更新，你说了算

### 5.2 基本要求

- 一台境外 VPS（推荐 Ubuntu 20.04+）
- 基本 Linux 命令行操作能力
- 1GB 内存即可（MTProto Proxy 资源占用极低）
- 建议同时设置防火墙规则，只开放必要端口

### 5.3 官方一键安装脚本

Telegram 官方提供了非常简洁的安装脚本：

```bash
# SSH 连接到 VPS 后执行
wget -O mtproto-install.sh https://raw.githubusercontent.com/TelegramMessenger/MTProxy/master/install.sh
chmod +x mtproto-install.sh
./mtproto-install.sh
```

安装完成后，脚本会输出代理连接信息，格式类似：

```
Proxy secret: ee443a2b...
Tag: 6a12...
```

把这个信息发给需要的人即可。对方在 Telegram 中添加 MTProto 代理时填入这些信息就能连接。

### 5.4 安全加固建议

搭建完成后，建议做以下加固：

- **使用非标准端口：** 不要用默认端口（如 443），用 8443、2053 等不那么明显的端口
- **限制连接来源：** 用 iptables/firewalld 限制访问来源 IP（如果你只给少数人使用）
- **定期更新：** `mtproto-proxy` 官方会发安全更新，记得 `git pull` 然后重启服务
- **监控异常流量：** 如果代理被人恶意扫描或滥用，VPS 流量会异常飙升

## 六、MTProto 的安全性争议与客观评价

### 6.1 密码学社区的质疑

自 MTProto 推出以来，密码学专家提出了几点关切：

**非标准加密模式。** MTProto 使用的 AES-256-IGE（Infinite Garble Extension）模式并非 AES 的标准工作模式（标准模式有 CBC、GCM、CTR 等）。Telegram 的解释是 IGE 对篡改更敏感，但这个选择增加了协议审查的难度。

**缺少独立的正式安全证明。** Signal Protocol 有多篇学术论文证明其安全性，MTProto 则缺少同等水平的学术背书。

**早期版本的已知问题。** MTProto 1.0 曾被发现 SHA-1 碰撞漏洞（CVE-2015-7862）。虽然 2.0 已修复，但这个历史影响了一些人对协议的信任。

### 6.2 MTProto 的辩护点

**实战检验超过十年。** 从 2013 年至今，MTProto 保护了全球数亿用户的通信安全。如果协议有根本设计缺陷，早就被大规模利用了。

**持续的第三方审计。** Telegram 多次组织独立安全审计，发包给知名密码学团队审查协议实现。已公开的审计报告显示没有致命漏洞。

**开放性。** Telegram 客户端完全开源，MTProto 的实现代码可以在 GitHub 上被任何人审查。

### 6.3 客观结论

MTProto 不是一个完美无缺的协议，但它是一个在实践中证明有效的协议。对于抵御大规模监控和流量分析来说，它提供了足够的安全保障。当然，如果你需要最高级别的安全（告密者、法律保护、国家安全级威胁），Signal Protocol 在理论上更受学术界认可。

关于不同工具的安全对比，建议阅读 [Telegram vs WhatsApp vs Signal 全面对比](/telegram-vs-whatsapp-vs-signal) 一文。

## 七、常见问题 FAQ

<details>
<summary>MTProto 代理和 VPN 有什么区别？</summary>

VPN 把所有网络流量（浏览器、邮件、App）都通过隧道发送，保护的是整个设备的所有通信。而 MTProto 代理只代理 Telegram 的流量，其他 App 不受影响。VPN 的加密层在 MTProto 之下——意味着 VPN 服务商可以知道你用了 Telegram（流量特征），但看不到 Telegram 的消息内容（MTProto 加密）。如果你只需要安全访问 Telegram，MTProto 代理比 VPN 更高效且更不容易被服务商分析。

</details>

<details>
<summary>使用 MTProto 代理后，我的 IP 地址会被对方看到吗？</summary>

不会。在 Telegram 通话或秘密聊天中，对方看到的 IP 地址取决于你的代理设置。如果你通过 MTProto 代理连接，对方看到的是代理服务器的 IP，不是你本机的真实 IP。如果你和对方在同一个 MTProto 代理后连接，Telegram 通常会尝试建立 P2P 直连（可能会暴露真实 IP），但这可以通过在「隐私与安全」设置中关闭「点对点」选项来避免。

</details>

<details>
<summary>MTProto 代理会被破解吗？</summary>

「破解」是一个需要精确理解的概念。MTProto 代理的加密基于 AES-256，使用目前公开已知的计算能力无法暴力破解。但以下情况可能导致安全性受损：(1) 代理服务器被攻破，攻击者获取了 auth_key（相当于拿到了加密钥匙）；(2) 量子计算成熟后可能对 DH-2048 构成威胁，但那是至少十年后的事。目前的实用威胁主要是代理服务器本身的物理安全和网络安全管理。

</details>

<details>
<summary>同一个 MTProto 代理可以给多人使用吗？</summary>

可以。MTProto 代理天然支持多用户。但需要注意：(1) 用户越多，代理服务器的带宽和 CPU 消耗越大；(2) 流量越大，越容易被网络管理者注意到并封锁 IP；(3) 如果其中一个用户发送了大量垃圾消息导致代理 IP 被 Telegram 封禁，所有用户都会受影响。建议小范围分享，不要公开发布代理地址。

</details>

**总结：** MTProto 是 Telegram 的核心技术资产。它在移动网络的特殊需求和安全加密之间找到了一个平衡点。虽然密码学社区对自研协议始终有保留意见，但十年的实战检验已经给了它足够强的可信度背书。

对于普通用户来说，理解 MTProto 的最大实用价值在于：知道如何通过 [MTProto代理](/) 安全访问 Telegram，以及明白即使通过代理，你的消息内容仍然是端到端加密保护的。如果你打算自己搭建代理，记住：安全不仅在于协议本身，更在于你如何管理服务器和分发密钥。
